# Core UX principles

Authoritative reference for Nielsen Norman’s **10 usability heuristics**: [10 Usability Heuristics for User Interface Design](https://www.nngroup.com/articles/ten-usability-heuristics/).

## Nielsen’s 10 heuristics (summary)

1. **Visibility of system status** — Show what is happening (loading, success, errors).
2. **Match between system and real world** — User language, familiar concepts, not jargon.
3. **User control and freedom** — Undo, cancel, escape hatches; easy recovery from mistakes.
4. **Consistency and standards** — Internal consistency; follow platform conventions.
5. **Error prevention** — Prefer constraints, confirmations, and validation over error messages alone.
6. **Recognition rather than recall** — Visible options, defaults, hints; minimize memory load.
7. **Flexibility and efficiency** — Shortcuts, bulk actions, customization for experts.
8. **Aesthetic and minimalist design** — Every element earns its place; reduce noise.
9. **Help users recognize, diagnose, and recover from errors** — Plain language, what happened, what to do next.
10. **Help and documentation** — Short, contextual, easy to search; not walls of text.

For deeper reading: [Nielsen Norman Group — UX research and guidelines](https://www.nngroup.com/).

## User-centered design

- Ground decisions in **tasks and context** (who, when, device, stress).
- Prefer **observed behavior** and feedback over assumptions.
- **Iterate**: sketch → test → refine; small experiments beat big bets without validation.

## Persona template (bullets)

- **Name / role** (label, not a biography).
- **Goals** (what they need to accomplish).
- **Frustrations** (what blocks them today).
- **Context** (devices, environment, skill level, time pressure).
- **Implications for product** (what to design for / avoid).

## User journey template (bullets)

- **Stage** (e.g. discover → consider → act → follow-up).
- **User action** (what they do).
- **Touchpoint** (where).
- **Emotion** (e.g. anxious, neutral, confident).
- **Pain points** (optional).
- **Opportunities** (what to improve next).

Use journeys to prioritize **critical paths** and **failure points**, not to document every screen.
